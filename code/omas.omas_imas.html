Browse available IMAS data (machine/pulse/run) for given user

Parameters:
  • user – user (of list of users) to browse. Browses all users if None.
  • pretty – express size in MB and time in human readeable format
  • quiet – print database to screen
  • user_imasdbdir – directory where imasdb is located for current user (typically $HOME/public/imasdb/)
Returns:

hierarchical dictionary with database of available IMAS data (machine/pulse/run) for given user

Taverse an IDS and list leaf paths (with proper sizing for arrays of structures)

Parameters:
  • ids – input ids
  • ds – hierarchical data schema as returned for example by load_structure(‘equilibrium’)[1]
  • requested_paths – list of paths that are requested
  • assume_uniform_array_structures – assume that the first structure in an array of structures has data in the same nodes locations of the later structures in the array
  • skip_ggd – do not traverse ggd structures
Returns:

returns list of paths in an IDS that are filled
